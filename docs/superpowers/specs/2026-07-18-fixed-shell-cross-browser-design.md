# Фиксированная кроссбраузерная оболочка без прыжков

## Цель

Полностью заменить текущую архитектуру страницы «Объявления скомпонованные»,
в которой корневая прокрутка связана с `transform: scale(...)` и динамической
высотой документа. Новая версия должна сохранять физический размер макета при
обычном browser zoom, не менять положение прокрутки и корректно работать в
Chromium-браузерах и Firefox, включая Firefox «Увеличивать только текст».

## Явное ограничение Firefox «Только текст»

Firefox принудительно увеличивает настоящий HTML-текст в режиме «Увеличивать
только текст». Страница не может стандартизованным способом запретить это, не
заменив текст Canvas/SVG-графикой. Поэтому утверждённое поведение такое:

- при обычном `Ctrl +/-` физический размер интерфейса компенсируется;
- при Firefox «Только текст» шрифт увеличивается по команде браузера;
- строки и контролы расширяются в нормальном потоке, ничего не перекрывается;
- внутренняя прокрутка не сбрасывается и не корректируется JavaScript;
- текст остаётся настоящим HTML: выделяемым, доступным и индексируемым.

## Архитектура оболочки

### Два независимых уровня

1. `#appHost` занимает видимую область окна, имеет `position: fixed`,
   `inset: 0` и `overflow: hidden`. Корневые `html` и `body` не прокручиваются.
2. `#appShell` содержит дизайн шириной 1920px и получает единственный
   `transform: scale(...)` с `transform-origin: 0 0`.
3. `#appScroll` внутри оболочки является единственным вертикальным
   scroll-контейнером. Его `scrollTop` никогда не записывается кодом масштаба.

Начальный `fitScale` вычисляется после готовности локальных шрифтов:

```text
fitScale = min(1, viewportWidth / 1920)
```

При обычном browser zoom итоговый коэффициент равен:

```text
shellScale = fitScale × baseDPR / currentDPR
```

Масштаб применяется только к `#appShell`. Размер `#appScroll` задаётся в
дизайн-пикселях и не зависит от высоты содержимого. Поэтому изменение масштаба
не меняет scroll range через высоту корневого документа и не требует
`window.scrollTo` или записи `scrollTop`.

### События окна

- `resize`, `visualViewport.resize` и изменение resolution media query
  планируют одно обновление через `requestAnimationFrame`.
- DPR-only событие меняет только `shellScale` и размеры оболочки.
- Реальный resize окна обновляет базовые размеры и `fitScale`.
- Обработчик не читает и не пишет позицию прокрутки.
- `ResizeObserver` разрешён только для измерения доступной области оболочки,
  но не для изменения высоты документа по высоте таблицы.

## Новая HTML-вёрстка

### Общие правила

- Чистые HTML, CSS и vanilla JavaScript без фреймворков и сборщика.
- Настоящий текст и семантические элементы управления.
- Локальные Roboto и `vsevn-tabs`; внешние запросы отсутствуют.
- Абсолютное позиционирование допускается только для декоративных иконок,
  underline и tooltip, но не для основных блоков страницы.
- Все видимые рамки имеют фактическую толщину `1px` и полупрозрачный цвет,
  имитирующий тонкие линии Figma без исчезновения на zoom-out.

### Шапка и фильтры

- Два ряда жёлтой шапки строятся CSS Grid/Flex.
- Вкладки источников остаются в порядке ТРУДВСЕМ, HH.RU, ZP.RU, RABOTA.RU,
  ВКОНТАКТЕ, ОК, ТЕЛЕГРАМ, МАКС.
- Для сайтов показываются «Модерация» и «Заблокированные на сайте».
- Для соцсетей эти статусы скрываются, а второй селект показывает
  «Все сообщества», «Все группы ОК», «Все каналы» или «Все чаты».
- Тулбар, радиобаттоны, чекбоксы и селекты находятся в нормальном потоке и
  могут переноситься при Firefox «Только текст» без наложений.

### Таблица

- Сохраняется семантический `<table>` и фиксированный `colgroup` для базовой
  геометрии 1920px.
- Ширины колонок соответствуют исходным границам Figma:
  `[50,127,269,639,732,825,918,1011,1104,1182,1470]`.
- Строки имеют автоматическую высоту; текст не фиксируется абсолютными
  координатами по вертикали.
- Заголовок и строка поиска остаются в таблице; ручной resize колонок
  обновляет ширины `colgroup`.
- Чередование строк, статусы, ссылки, подсветка поиска и режимы
  «Переносить», «Свернуть все», «Без тегов» сохраняются.

## Разделение файлов

- `css/base.css` — шрифты, reset, палитра, `appHost/appShell/appScroll`.
- `css/components.css` — шапка, вкладки, фильтры, кнопки, селекты, календарь.
- `css/table.css` — таблица, колонки, строки, режим увеличенного текста.
- `js/viewport.js` — только вычисление и применение масштаба оболочки.
- `js/app.js` — состояние, данные, фильтрация, рендер и взаимодействия.
- `js/data.js` — демонстрационные данные без изменений формата.

`viewport.js` предоставляет объект `window.AppViewport`:

```text
init(): запускает оболочку после готовности шрифтов
refresh(): пересчитывает масштаб без доступа к scrollTop
getScale(): возвращает текущий коэффициент для tooltip-координат
```

`app.js` не подписывается на `resize` и не управляет размерами оболочки.

## Сохраняемая функциональность

- Переключение источников и режимов сайт/соцсеть.
- Поиск с начала слова без регистра, дефисов и кавычек.
- Поиск номера счёта с игнорированием нулей после букв.
- Сортировка А–Я/Я–А, фильтры, календарь и чекбоксоподобные радио.
- Пагинация, выбор 25/50/100/250/500/1000 строк.
- Экспорт XML/XLS, копирование ссылок и tooltip `#DFD9D9`.
- Ручное изменение ширины колонок.

## Проверка

### Автоматические контракты

1. В `viewport.js` отсутствуют `window.scrollTo`, записи `scrollTop` и чтение
   высоты таблицы для изменения документа.
2. `html` и `body` не прокручиваются; `#appScroll` — единственный элемент с
   `overflow-y: auto` и реальным вертикальным overflow.
3. `app.js` не содержит resize/DPR-логики.
4. Все видимые рамки используют `1px`, а не `0.3px`.
5. `node --check` проходит для `viewport.js`, `app.js` и `data.js`.

### Браузерные сценарии

- Chromium: `80% → 100% → 125% → 150% → 100%` при прокрутке внутри таблицы.
  `appScroll.scrollTop` должен оставаться численно неизменным.
- Реальный resize окна: оболочка вписывается заново без сброса прокрутки.
- Имитация text-only `125%, 150%, 200%`: увеличиваются только font-size и
  line-height; строки расширяются, элементы не перекрываются.
- Вкладки, фильтры, поиск, сортировка и пагинация проходят smoke-тест без
  ошибок и предупреждений консоли.
- Firefox «Увеличивать только текст» проверяется вручную перед передачей;
  если Firefox недоступен в среде, это фиксируется явно.

## Границы итерации

Вёрстка и viewport-архитектура создаются заново. Формат данных и бизнес-логика
поиска, фильтров, вкладок и экспорта переносятся без изменения результата.
Pixel-perfect оценивается на 1920px при базовом масштабе; устойчивость к zoom и
Firefox text-only имеет приоритет над субпиксельными линиями и абсолютным
вертикальным позиционированием старой версии.
