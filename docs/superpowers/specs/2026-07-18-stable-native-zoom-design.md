# Стабильный нативный зум для «Объявления скомпонованные»

## Цель

Устранить вертикальные скачки страницы при `Ctrl +` / `Ctrl -`, сохранив
pixel-perfect геометрию артборда «Если выбрать hh» из
`Объявления_скомпанованное_общее_1.fig` и функциональные требования из
`Объявления ОК первая.docx`.

## Диагноз

Текущая сборка реагирует на каждый `resize` так:

1. определяет изменение `devicePixelRatio`;
2. меняет `transform: scale(...)` у 1920-пиксельного кадра;
3. пересчитывает высоту внешнего viewport;
4. вручную меняет `window.scrollY`.

При браузерном зуме движок уже меняет CSS viewport и сам корректирует
прокрутку. Дополнительный `scrollTo` из JS конкурирует с этой коррекцией.
`overflow-anchor: none` дополнительно отключает штатное удержание видимого
контента. В диагностическом прогоне переход DPR `1 → 1.25` изменил CSS
`scrollY` с `5000` на `4000`, масштаб кадра с `0.994792` на `0.794792`, а
высоту документа с `85906` до `68635` пикселей за один переход. Это и есть
источник наблюдаемого скачка.

## Принятое решение

Используется нативный зум браузера с обратной компенсацией только на кадре.
JavaScript не меняет масштаб на каждый промежуточный resize-сигнал и не
пересчитывает прокрутку каскадом.

### Координатная система

- Внутренний кадр остаётся шириной `1920` design-px.
- Все координаты, цвета, тексты и порядок колонок берутся из текущей
  сборки, сверенной с `.fig`.
- Масштаб вписывания вычисляется после загрузки локальных шрифтов:
  `fitScale = min(1, document.documentElement.clientWidth / 1920)`.
- При browser zoom сохраняется базовый `fitScale`, а кадр получает
  `fitScale * baseDPR / currentDPR`. Поэтому физический размер кадра и тонкие
  границы не уменьшаются при `Ctrl -` и не раздуваются при `Ctrl +`.
- Обновление масштаба и высоты выполняется один раз в `requestAnimationFrame`.
  На время этого кадра сохраняется видимый элемент в центре viewport; после
  применения масштаба его положение восстанавливается одним `scrollTo`, а не
  серией коррекций.
- При зуме выше 100% допускается горизонтальный overflow корневой страницы,
  но вложенных вертикальных scroll-контейнеров быть не должно.

### Реальный resize окна

Изменение размера окна — отдельный сценарий, не равный браузерному зуму.

- При инициализации сохраняются `outerWidth`, `outerHeight` и исходный DPR.
- Если DPR изменился, но размеры внешнего окна не изменились, событие
  считается браузерным зумом и игнорируется масштабировщиком.
- Если изменился `outerWidth` или `outerHeight`, через один
  `requestAnimationFrame` пересчитывается `fitScale` и высота viewport.
- При настоящем resize пересчитывается только базовый fitScale. При browser
  zoom якорь видимого элемента восстанавливается один раз в том же кадре,
  что и компенсация DPR; повторных `scrollTo` на цепочке resize нет.
- `ResizeObserver` используется только для изменения фактической высоты
  контента. Запись `style.height` выполняется только если значение реально
  изменилось, чтобы не создавать каскад layout/resize событий.

### Первый показ страницы и шрифты

- Локальные Roboto и `vsevn-tabs` остаются предзагруженными.
- До готовности `document.fonts.ready` кадр скрыт без изменения его геометрии.
- После готовности шрифтов выполняются ровно один раз: начальный рендер,
  расчёт `fitScale`, синхронизация высоты и позиционирование underline.
- Для аварийного случая добавляется ограниченный fallback, чтобы страница
  не оставалась скрытой при проблеме загрузки шрифта.

### Прокрутка и якорение

- Удаляется `overflow-anchor: none` с `html`.
- Для корневой страницы включается штатное `overflow-anchor: auto`.
- `body`, `#viewport` и таблица не получают собственных вертикальных
  overflow-контейнеров.
- Обработчик browser zoom не пишет `document.scrollingElement.scrollTop` и не
  пересоздаёт документ; допускается один `window.scrollTo` после изменения
  масштаба для восстановления захваченного экранного якоря.
- Рендер фильтров обновляет только `tbody`, как и сейчас; поведение перехода
  на первую страницу при изменении фильтра сохраняется.

## Что сохраняется из ТЗ

- Чистые HTML/CSS/vanilla JS без сборщика, jQuery и UI-библиотек.
- Две строки жёлтой шапки, иконочный шрифт вкладок и активные underline.
- Сайтовые вкладки показывают «Модерация» и «Заблокированные на сайте»;
  социальные вкладки скрывают их и показывают второй селект с плейсхолдером
  сети: «Все сообщества», «Все группы ОК», «Все каналы», «Все чаты».
- Поиск с начала слова без регистра, дефисов, кавычек и лишних нулей после
  букв в номере счёта; совпадения подсвечиваются голубым.
- Сортировка А–Я / Я–А, чекбоксоподобные радиобаттоны, селекты, календарь,
  пагинация, экспорт XML/XLS, копирование ссылок и подсказки `#DFD9D9`.
- Чередование белых строк и строк `#FFFEFB`, ручной resize колонок и режимы
  «Переносить», «Свернуть все», «Без тегов».

## Измеримые критерии приёмки

1. При загрузке на viewport 1920 px нормализованные координаты ключевых
   элементов совпадают с `.fig` с допуском не более 1 design-px. В частности:
   заголовок начинается около `(64,159)`, дата «от» — около `(68,222)`,
   селект XML — около `(71,403)`, а границы таблицы соответствуют колонкам
   `[50,127,269,639,732,825,918,1011,1104,1182,1470]`.
2. В последовательности зума `80% → 100% → 125% → 150% → 100%` физическая
   ширина кадра и видимая центральная ячейка остаются стабильными; на каждый
   переход выполняется не более одной коррекции якоря.
3. При зуме высота внешнего viewport не пересчитывается повторно, а число
   вертикальных scroll-контейнеров остаётся равным одному — корневому.
4. Контрольная строка таблицы не получает дополнительного скачка после
   завершения нативного зума: два замера через 100 мс после стабилизации
   отличаются не более чем на 1 CSS-px сверх изменения, внесённого самим
   браузером.
5. Переключение вкладок, фильтры и поиск по-прежнему меняют количество
   строк и не создают ошибок в консоли.
6. `node --check` проходит для `js/app.js` и `js/data.js`; автоматический
   браузерный smoke-тест проходит без console errors.
7. В Firefox и Chromium-подобном браузере визуальная проверка выполняется
   отдельно на 100% и при последовательности Ctrl +/-; отсутствие
   Firefox-среды в контейнере фиксируется явно, а не маскируется заявлением
   о полном кроссбраузерном тесте.

## Границы работ

В этой итерации исправляется масштабирование, якорение и связанные с ним
layout-shift причины. Содержательные требования таблицы и вкладок не
переписываются заново; любые найденные расхождения координат исправляются
точечно после стабилизации зума и сверяются с исходным `.fig`.
