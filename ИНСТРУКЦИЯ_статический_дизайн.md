# Инструкция: как реализован статический пиксель-перфект дизайн

Документ описывает принципы, по которым сделана вёрстка «Объявления скомпонованные»,
чтобы любой программист мог реализовать такой же статический дизайн по аналогии.

Стек: **чистый HTML + CSS + ванильный JavaScript**. Без библиотек, jQuery,
Bootstrap, фреймворков и сборщиков. Все файлы подключаются напрямую `<script>`/`<link>`.

---

## 1. Базовая идея масштабирования (пиксель-перфект по макету 1920px)

Макет Figma нарисован на артборде шириной **1920px**. Задача — чтобы вёрстка выглядела
попиксельно как макет, но при этом масштабировалась под ширину окна.

Приём: **все размеры задаются не в пикселях, а в долях от ширины экрана**, кратных
одному «дизайн-пикселю».

### CSS-переменные единиц масштаба

```css
/* 1% ширины «расчётной» области ≈ 1vw. Один дизайн-пиксель = --fvw / 19.2,
   потому что 1920px / 100 = 19.2px на 1% ширины. */
.ads-page {
  --px: var(--dpx, calc(var(--fvw, 1vw) / 19.2));
}
```

- `--fvw` — ширина 1% расчётной области в px. Задаётся из JS (`syncDesignViewportUnit`).
- `--dpx` — размер одного дизайн-пикселя в px (`pageScale`, где `pageScale = ширина / 1920`).
- `--px` — рабочая единица: `var(--dpx)` или запасной вариант `calc(var(--fvw)/19.2)`.

### Как задаются размеры

Любая величина из макета (в дизайн-пикселях) умножается на `--px`:

```css
.ads-card {
  width: calc(1881 * var(--px));      /* 1881 дизайн-px по макету */
  margin-left: calc(19 * var(--px));  /* отступ 19 дизайн-px */
}
.ads-table { font-size: calc(20 * var(--px)); }
```

В JS для того же есть хелпер `pxToVw(px)`:

```js
function pxToVw(px) {
  return "calc(var(--dpx) * " + Number(px).toFixed(5).replace(/\.?0+$/, "") + ")";
}
```

**Итог:** при любой ширине окна пропорции макета сохраняются точно — это и есть
пиксель-перфект относительно артборда 1920px.

## 2. Живой HTML-текст (запрещено запекать текст в SVG/картинки)

Требование заказчика: весь текст должен оставаться **настоящим HTML** — выделяемым,
копируемым, доступным для поиска и скринридеров. **Запрещено** превращать текст в
SVG или растровые изображения (частая ошибка: «запечь» текст в картинку, чтобы убрать
субпиксельные смещения — так делать нельзя).

Как сделано правильно:

- Подписи выводятся обычным текстом в элементы (`textContent`), а размер/начертание
  задаются инлайн-стилями через ту же единицу `--px`:

```js
function renderPlainElementText(element, text, options) {
  element.textContent = text;
  element.style.setProperty("font-size", pxToVw(options.size), "important");
  element.style.setProperty("line-height", pxToVw(options.height), "important");
  element.style.setProperty("font-weight", String(options.weight), "important");
}
```

- Центральная функция `renderElementText()` направляет все подписи по «живому» пути
  (никаких `<svg>` с текстом внутри). Проверка: выделите заголовок или подпись мышью —
  текст должен выделяться. На странице не должно быть `svg.svg-label` с текстом.
- SVG допустим **только для иконок** (стрелки сортировки, иконки навигации), но не для
  контентного текста.

## 3. Борьба с субпиксельными смещениями (без запекания в картинки)

Проблема: `--px` — дробная величина (например 0.9948px). Тогда `calc(N * var(--px))`
даёт дробные координаты, и границы соседних элементов (особенно линии таблицы) попадают
на «пол-пикселя» → браузер рисует их по-разному → рваная сетка и микросмещения.

Решение — **округление ключевых величин до целого физического пикселя** средствами CSS
(`round()`), без изменения общей концепции масштабирования:

```css
.ads-card {
  width: round(calc(1881 * var(--px)), 1px);        /* ширина к целому px */
  margin-left: round(calc(19 * var(--px)), 1px);    /* origin к целому px */
}
.ads-table-wrapper {
  padding-left: round(calc(18 * var(--px)), 1px);   /* origin таблицы к целому px */
}
```

Ширины колонок таблицы тоже округляются к целым пикселям (в `<colgroup>`):

```js
'<col style="width:round(calc(' + col.width + ' * var(--px)), 1px)">'
```

Так вертикальные границы ячеек становятся на равномерной пиксельной сетке — исчезают
случайные швы и микросмещения, при этом текст остаётся живым HTML.

> `round()` поддерживается современными браузерами (Chrome/Edge 112+, Safari 15.4+, Firefox 118+).

## 4. Поведение при браузерном зуме (Ctrl «+»/«−») — без «прыжков»

Главная ошибка предыдущих реализаций: при каждом шаге зума код **пересчитывал** масштаб
макета и пытался «нейтрализовать» зум, причём по неточной оценке величины зума. Из-за
этого на каждый шаг зума текст чуть по-разному раскладывался — визуально «прыгал».

Правильное поведение — **отдать зум браузеру нативно**:

1. Не ставить `zoom`/`transform` на контейнер, чтобы «догнать» браузерный зум.
2. Не пересчитывать `--dpx`/`--fvw` во время зума — тогда браузер просто масштабирует
   уже готовую страницу целиком, и ничего не перекладывается.

Зум надёжно отличается от ресайза окна по изменению `devicePixelRatio`
(браузерный зум его меняет, ресайз окна — нет):

```js
let baselineDeviceRatio = null;              // зафиксировали на загрузке
function isBrowserZoomed() {
  return Math.abs(devicePixelRatio / baselineDeviceRatio - 1) > 0.02;
}
```

В функциях пересчёта метрик — ранний выход при зуме:

```js
function updateZoomAwareLines() {
  if (isBrowserZoomed()) return;   // при зуме масштаб НЕ трогаем
  /* ... пересчёт --dpx только при обычном состоянии/ресайзе ... */
}
```

Результат: при Ctrl «+»/«−» страница просто плавно приближается/отдаляется, как обычный
сайт, без дёрганья и без перекладки текста. Пиксель-перфект сохраняется, т.к. всё
масштабируется пропорционально.

## 5. Подсказки (tooltip) только при реальном наведении мышью

Ошибка: подсказки на чистом CSS `:hover` всплывали сами при зуме — когда элемент
«подъезжал» под неподвижный курсор из-за перекладки лейаута.

Решение — показывать подсказку только при **осознанном** наведении (было реальное
движение мыши), а на изменении вьюпорта сбрасывать это состояние:

```js
let hoverIntentActive = false;
function setHoverIntent(on) {
  document.documentElement.classList.toggle("has-hover-intent", on);
  hoverIntentActive = on;
}
window.addEventListener("pointermove", () => setHoverIntent(true), true);
function markViewportChanging() { setHoverIntent(false); /* ... */ }
```

CSS показывает подсказку только при наличии класса:

```css
html.has-hover-intent .icon-wrapper:hover > .tooltip { display: flex; }
```

Браузерный зум не порождает `pointermove` (курсор физически не двигается) → подсказка
сама не всплывает.

## 6. Шрифты — только локальные (без внешних запросов)

Для мгновенного вывода контента и самодостаточности шрифт подключается **локально**
через `@font-face` (файлы `.woff2` в `/fonts`), без обращения к Google Fonts:

```css
@font-face {
  font-family: 'Roboto';
  font-weight: 100 900;                 /* один вариативный шрифт на все начертания */
  font-display: block;                  /* без «мигания» подменного шрифта */
  src: url('../fonts/roboto-cyrillic.woff2') format('woff2');
  unicode-range: U+0400-045F, ...;      /* кириллический сабсет */
}
```

Основной сабсет предзагружается, чтобы текст появлялся сразу:

```html
<link rel="preload" href="fonts/roboto-cyrillic.woff2" as="font" type="font/woff2" crossorigin />
```

Никаких `<link>` на внешние CDN — страница работает офлайн и грузится мгновенно
(нет render-blocking внешних запросов).

## 7. Одна вертикальная прокрутка (без двойного ползунка)

Прокрутка организована на одном контейнере (`html`); вложенные контейнеры (`body`,
`#zoomFrame`) не создают собственный вертикальный скролл — за счёт этого нет двух
ползунков:

```css
html  { overflow-y: scroll; overflow-x: hidden; scrollbar-gutter: stable; }
body  { overflow: hidden; }
#zoomFrame { overflow-x: hidden; }
```

`scrollbar-gutter: stable` резервирует место под полосу прокрутки, чтобы ширина
контентной области не «дёргалась» при появлении/исчезновении скролла.

## 8. Данные и структура (data-driven)

Колонки, цвета и раскладка вынесены в data-файлы и рендерятся ванильным JS:

- `js/data/figma-columns.js` — описание колонок таблицы (ключ, ширина в дизайн-px, заголовок).
- `js/data/figma-colors.js`, `figma-layout.json`, `figma-spec.json` — цвета и геометрия из Figma.
- `js/data/ads-data.js` — данные объявлений.
- `js/script.js` — общий движок (единицы масштаба, живой текст, зум, подсказки, календарь).
- `js/compiled-ads-app.js` — логика страницы объявлений (таблица, фильтры, сортировка, экспорт).

Порядок подключения важен (данные → движок → страница):

```html
<script src="js/data/figma-columns.js"></script>
<script src="js/data/ads-data.js"></script>
<script src="js/script.js"></script>
<script src="js/compiled-ads-app.js"></script>
```

## 9. Чек-лист для повторения статического дизайна

1. Возьмите ширину артборда макета (здесь 1920px), задайте единицу
   `--px = ширинаЭкрана / артборд`.
2. Все размеры пишите как `calc(N * var(--px))`, где N — величина из макета в дизайн-px.
3. Текст выводите живым HTML (`textContent` + font-size через `--px`), **не запекайте в SVG**.
4. Ключевые координаты/ширины (карточка, колонки, origin таблицы) округляйте до целого
   пикселя через `round(calc(...), 1px)` — уберёт субпиксельные швы.
5. Браузерный зум не перехватывайте и не пересчитывайте — отдайте браузеру нативно;
   пересчёт метрик замораживайте при `isBrowserZoomed()`.
6. Подсказки показывайте только при реальном `pointermove`, сбрасывая на изменении вьюпорта.
7. Шрифты — локальный `@font-face` + `preload`, без внешних CDN.
8. Одна прокрутка на `html`, вложенные контейнеры без собственного вертикального скролла.
9. Никаких библиотек/сборщиков — только прямые `<link>`/`<script>`.
